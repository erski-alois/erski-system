# ERSki 滑雪急診室 預約系統 — 開發版 v6

這次是版面大改版:黑色版頭、右側巢狀導覽(室內滑雪/日本滑雪各有下拉子項目)、新增
「教練團隊」分頁、日本滑雪改成 5 個分區(藏王/北海道/鬼首/白馬/其他雪場,各自規則不同)、
每一個預約流程都改成「先選裝備(雙板/單板)→ 選人數 → 填每個人的性別/年齡/身高/體重/
鞋碼 → 最後選日期時間」。

## 版面結構

- **黑色版頭**,標題與會員資訊同一行
- **右側導覽**(桌面版在右邊,手機版會自動變成上方):
  - 室內滑雪(展開子項目,依優先順序):包機預約 → 團課預約 → 體驗課預約 →
    自主練習預約 → 跳台預約
  - 日本滑雪(展開子項目):藏王溫泉滑雪場、北海道滑雪場、鬼首滑雪、白馬地區滑雪場、
    其他雪場
  - 教練團隊
  - 會員中心

## 每個預約流程的新順序

所有預約(體驗課/包機/自主練習/團課/跳台/日本教練課)都改成:

1. 選裝備:**雙板 SKI** 或 **單板 Snowboard**
2. 選人數(團課固定1人,其餘視課程可選 1~4 人或 1~2 人)
3. 依人數自動產生每個人的表單:**性別、年齡、身高、體重、鞋子尺寸**
4. 最後才選日期與時段

## 日本滑雪 5 個分區規則

| 分區 | 需選雪場 | 可指定教練 | 需詢問住宿 | 雪場清單可自行增刪 |
|---|---|---|---|---|
| 藏王溫泉滑雪場 | 否 | 是 | 否 | 否 |
| 北海道滑雪場 | 是(固定5個:手稻/札幌國際/朝里川/Kiroro/Onze) | 是 | 否 | 否 |
| 鬼首滑雪 | 否 | 是 | **是** | 否 |
| 白馬地區滑雪場 | 是 | 是 | 否 | **是** |
| 其他雪場 | 是 | **否(只能由校方指派)** | 否 | **是** |

雪場清單開放新增的分區(白馬、其他雪場),前台雪場選單旁邊有「管理此分區雪場清單」按鈕,
目前是簡易版(用瀏覽器輸入框新增),之後可以做成更完整的後台介面。

## 教練團隊

新增公開的「教練團隊」頁面,顯示所有教練的宣傳照與自我介紹。後台可透過
`PUT /api/admin/coaches/<id>/profile` 上傳宣傳照(圖片轉 base64 文字)、自我介紹、
證件照(僅供內部使用,不對外顯示)。

> ⚠️ 這裡有個技術限制要跟你說清楚:圖片目前是直接存成文字(base64)放進資料庫,
> 沒有串接真正的雲端圖床。小張大頭照沒問題,但如果照片檔案很大(例如手機拍的原始照片
> 好幾MB),資料庫會變得很肥大、載入變慢。等你要正式上線,建議串接真正的圖片空間
> (例如 Cloudinary、AWS S3),這部分需要有網路的環境才能設定,可以之後再協助你規劃。

目前這版本教練團隊的照片上傳還沒有做前台的上傳按鈕介面(只有 API),下一步可以幫你做。

## 已實作並測試通過的規則(累計)

- 預約時間:不限制未來天數;日本滑雪固定12月-隔年4月雪季開放
- 室內機台:同一時段僅一筆有效預約;跳台獨立資源不受限
- 團課:滿2人開課滿4人截止、不可指定教練、方案由後台指派(客戶不能自選)、每時段顯示即時人數
- 日本教練課:先選雪場再選教練(整趟行程共用)、依天數自動統計總價、教練衝堂檢查、
  不指定教練時依雪場教練名單計算容量避免 over booking、教練出勤含請假/出差
- 會員中心:完整個人資料表單(會員自填)、帳戶總覽含已預約/未完成課程與教練姓名
- 每筆預約都記錄裝備類型與每位參與者的個人資料(性別/年齡/身高/體重/鞋碼)

## 如何在你的電腦上執行

```bash
cd backend
pip3 install -r requirements.txt
python3 db.py       # 初始化資料庫(會自動建立示範員工帳號、雪場分區資料)
python3 app.py       # 啟動,http://127.0.0.1:5001
```

## 部署到 Render

跟之前一樣,GitHub push 後在 Render 對現有服務按 Manual Deploy 重新部署即可。
